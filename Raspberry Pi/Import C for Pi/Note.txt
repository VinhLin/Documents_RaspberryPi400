- Khi ta viết code Go mà có phần import "C" thì khi build để chạy trên Raspberry thì nó sẽ báo lỗi.

- Sau khi tìm một cách giải quyết thì mình có làm theo hướng dẫn của 2 trang này: 
> https://www.raspberrypi.org/documentation/linux/kernel/building.md
> https://github.com/abhiTronix/raspberry-pi-cross-compilers/tree/master/build-scripts
- Có điều mình không biết là do mình chỉ cần cài đặt 1 trong 2 rồi chạy lệnh như bên dưới, hay là mình phải cài đặt cả hai mới có thể build đc. Nếu cài đặt cả 2 thì tốn dung lượng lên đến hơn 4G và mất hơn 3 tiếng để cài đặt.
	> GOOS=linux GOARCH=arm CGO_ENABLED=1 CC=arm-linux-gnueabihf-gcc go build -o test maintest.go

- Các bước cài đặt của mình (lưu ý là các bước cài đặt mình đều thực hiện trên PC):
#### Theo hướng dẫn link 1 (cài đặt cái này mất hơn 2 tiếng đồng hồ):
	> git clone https://github.com/raspberrypi/tools ~/tools
	> echo PATH=\$PATH:~/tools/arm-bcm2708/arm-linux-gnueabihf/bin >> ~/.bashrc
	> source ~/.bashrc
	> sudo apt install zlib1g-dev:amd64
	> git clone --depth=1 https://github.com/raspberrypi/linux
	> sudo apt install git bc bison flex libssl-dev make libc6-dev libncurses5-dev
	> cd linux
	> KERNEL=kernel7
	> make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- bcm2709_defconfig
	> make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- zImage modules dtbs
#### Theo hướng dẫn link 2 (cài đặt cái này có vẻ nhanh hơn):
	> sudo apt update && sudo apt dist-upgrade
	> sudo apt-get -y install gcc g++ gperf flex texinfo gawk gfortran texinfo bison \
    build-essential openssl unzip wget git pigz \
    libncurses-dev autoconf automake tar figlet
	> sudo apt install -y ccache
	> sudo /usr/sbin/update-ccache-symlinks
	> echo 'export PATH="/usr/lib/ccache:$PATH"' | tee -a ~/.bashrc
	> export CCACHE_COMPRESS=1
	> export CCACHE_DIR=$HOME/.ccache
	> source ~/.bashrc && echo $PATH
	> git clone https://github.com/abhiTronix/raspberry-pi-cross-compilers.git
	> cd raspberry-pi-cross-compilers/build-scripts
	> chmod +x RTBuilder_64b
	> ./RTBuilder_64b -g "9.2.0" -t "buster"
